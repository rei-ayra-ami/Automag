# Запуск приложения Automag

Теперь приложению **не нужен отдельный сервер** (Python/FastAPI). Все данные —
товары, пользователи и заказы — хранятся во **встроенной базе данных** прямо
внутри приложения (SQLite через пакет `drift`). База создаётся и наполняется
автоматически при первом запуске.

## Что нужно один раз

Установленный Flutter (проект собран на Flutter 3.27 / Dart 3.6).
Проверить: `flutter doctor`.

## Запуск — две команды

```bash
flutter pub get        # подтянуть зависимости (один раз после git pull)
flutter run            # запустить приложение
```

Выбрать устройство при запросе (или сразу):

```bash
flutter run -d chrome   # в браузере Chrome
flutter run -d linux    # как настольное приложение
```

> Первый запуск может занять чуть больше времени — создаётся база данных
> и в неё заносится каталог из 37 товаров.

## Демо-аккаунт администратора

Заводится автоматически при первом запуске:

- **Email:** `admin@automag.kz`
- **Пароль:** `admin123`

Под этим аккаунтом во вкладке **«Профиль»** появляются разделы:
- **Управление заказами** — все заказы магазина: кто оформил, на какую сумму,
  смена статуса (Новый → В обработке → Отправлен → Доставлен / Отменён).
- **Добавить товар** — регистрация новой запчасти в каталоге.

Обычные покупатели регистрируются сами через экран **«Регистрация»**.

## Что нового

- **База данных** с таблицами: пользователи, товары, заказы, позиции заказа,
  история статусов.
- **Регистрация и вход** теперь через базу (пароли хранятся в виде хеша, а не
  открытым текстом). Можно завести несколько разных аккаунтов.
- **Оформление заказа** сохраняет заказ в базу. История доступна в разделе
  **«Профиль» → «Мои заказы»** вместе с текущим статусом.
- **Отслеживание заказов**: видно, кто что заказал, и в каком заказ статусе —
  как в обычных интернет-магазинах.

## Где лежит база данных

- **Desktop/мобильное приложение:** файл `automag` в системном каталоге данных
  приложения.
- **Web (Chrome):** хранилище браузера (IndexedDB).

Чтобы начать с чистой базы (заново создать товары и админа), достаточно удалить
данные приложения/очистить хранилище сайта.

## Про прежний сервер FastAPI

Папка `backend/` оставлена в проекте для справки, но **для работы приложения
больше не требуется**. Запускать `uvicorn`/venv не нужно.
